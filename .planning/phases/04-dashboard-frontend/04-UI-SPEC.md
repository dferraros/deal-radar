## Spacing Scale

Declared values (multiples of 4 only). All spacing implemented via Tailwind utility classes.

| Token | Value | Tailwind class | Usage |
|-------|-------|---------------|-------|
| xs | 4px | gap-1, p-1 | Icon gaps, badge padding |
| sm | 8px | gap-2, p-2, px-2 | Compact inline spacing, filter tag gaps |
| md | 16px | gap-4, p-4, px-4 | Default card padding, element spacing |
| lg | 24px | gap-6, p-6, mb-6 | Section padding, card body |
| xl | 32px | gap-8, mb-8 | Layout column gaps |
| 2xl | 48px | py-12 | Major section breaks between KPI row and table |
| 3xl | 64px | — | Not used in this phase (no hero sections) |

Exceptions:
- Nav height: 56px (h-14) — locked from existing Navbar.tsx implementation
- Page container max-width: 1280px (max-w-7xl), horizontal padding px-4 sm:px-6 lg:px-8 — locked from App.tsx
- Touch targets on watchlist remove button: minimum 44px height (h-11)

## Typography

All sizes in px. Font family: Tailwind default sans (Inter/system-ui). Tabular numerics on all monetary and numeric columns via `tabular-nums` class.

| Role | Size | Tailwind | Weight | Tailwind | Line Height |
|------|------|----------|--------|----------|-------------|
| Body | 14px | text-sm | 400 regular | font-normal | 1.5 (leading-normal) |
| Label | 12px | text-xs | 500 medium | font-medium | 1.4 (leading-tight) |
| Heading | 20px | text-xl | 700 bold | font-bold | 1.2 (leading-tight) |
| Display | 28px | text-2xl | 700 bold | font-bold | 1.2 (leading-tight) |

Rules:
- Page titles (Deal Feed, Heatmap, etc.): Display — 28px bold, `text-gray-100`
- KPI card metric values: Heading — 20px bold, `text-gray-100`, tabular-nums
- KPI card labels: Label — 12px medium, `text-gray-400`
- Table column headers: Label — 12px medium, `text-gray-400 uppercase tracking-wide`
- Table cell data: Body — 14px regular, `text-gray-100` for primary, `text-gray-400` for secondary/muted
- Filter bar labels: Body — 14px regular, `text-gray-400`
- Empty state headings: Heading — 20px bold, `text-gray-300`
- Empty state body: Body — 14px regular, `text-gray-400`

**Source:** Existing Navbar.tsx + DealFeed.tsx stub patterns + design_context block.

## View-by-View Interaction Contract

### View 1: Deal Feed (/)

**Layout:**
- KPI row: 3 Tremor KPI cards in a 3-column grid (grid-cols-1 sm:grid-cols-3), gap-4
  - "Deals This Week" | "Capital Raised (7d)" | "Top Sector"
- Filter bar: horizontal flex row, gap-3, below KPI row, mb-6
  - Tremor Select: Deal Type (All / VC / M&A / Crypto / IPO)
  - Tremor Select: Sector (All + distinct sectors from API)
  - Tremor Select: Geo (All / LatAm / Spain / Europe / Global)
  - Tremor NumberInput or text input: Min Amount (USD)
  - Tremor DateRangePicker: Date range (default: last 7 days)
  - "Clear filters" text button, text-amber-400, only visible when any filter is active
- Tremor Table: full-width, below filter bar
  - Columns (in order): Date | Company | Round | Amount | Sector | Geo | Investors
  - Date: text-sm text-gray-400, format "Apr 04"
  - Company: text-sm font-medium text-gray-100
  - Round: Tremor Badge with deal type color (see badge color table above) + round_label text
  - Amount: text-sm text-gray-100 tabular-nums, "Undisclosed" in text-gray-400 italic when null
  - Sector: text-sm text-gray-400, comma-separated if multiple
  - Geo: text-sm text-gray-400
  - Investors: text-sm text-gray-400, first investor name only + "+N more" if >1
  - Row click: navigate to /company/:id (entire row is clickable, cursor-pointer, hover:bg-gray-800)
- Loading state: Tremor skeleton or spinner centered in table area
- Pagination: "Load more" button at bottom (text-sm text-amber-400 underline), not numbered pagination

**Interactions:**
- Filter change: re-fetch /api/deals with updated query params, show loading state on table only
- Row click: navigate to /company/:id
- "Clear filters": reset all filter state to defaults, re-fetch

### View 2: Sector Heatmap (/heatmap)

**Layout:**
- Page title: "Sector Heatmap" (Display, mb-2)
- Subtitle: "Capital raised by sector and geography" (Body, text-gray-400, mb-6)
- Period toggle: Tremor SegmentedControl — "Weekly" | "Monthly" | "Quarterly", right-aligned above grid, mb-4
- Grid: CSS grid, rows = sectors (Y axis), cols = geo labels (X axis)
  - Header row: geo labels ("LatAm", "Spain", "Europe", "Global"), text-xs text-gray-400 uppercase, text-center
  - Row label column: sector names, text-xs text-gray-400, text-right, pr-3, w-24
  - Cells: min-w-16 h-12, rounded-sm, relative
  - Cell hover: show Tremor Tooltip — "{sector} / {geo}: ${amount}M · {count} deals"
  - Cell with zero/no data: bg-gray-800 (no tooltip needed, or tooltip shows "No deals")
  - Color scale: see Color section above (gray-800 → amber-600)

**Interactions:**
- Period toggle change: re-fetch /api/heatmap?period={value}, replace grid data
- Cell hover: show tooltip immediately (no delay)

### View 3: Company Profile (/company/:id)

**Layout:**
- Back link: "← Deal Feed" at top, text-sm text-amber-400, navigate(-1), mb-4
- Header card (Tremor Card, bg-gray-900):
  - Company name: Display (28px bold, text-gray-100)
  - Sector badges: Tremor Badge, color="blue", one per sector tag, gap-2, mt-2
  - Geo: text-sm text-gray-400, mt-1
  - Website: text-sm text-amber-400 underline, external link (target="_blank" rel="noopener"), mt-1
  - Description: Body (14px, text-gray-300), mt-4, max 3 lines then truncate with "Show more" toggle
  - Watchlist button: top-right of card, Tremor Button variant="secondary"
    - Not watchlisted: "Add to Watchlist" with star icon (outline), color default
    - Watchlisted: "On Watchlist" with star icon (filled, text-amber-400), color="amber" variant
    - Click: optimistic toggle — update UI immediately, then POST/DELETE /api/watchlist

- Deal history section:
  - Section heading: "Deal History" (Heading, 20px bold, text-gray-100, mt-8 mb-4)
  - Tremor List, each item:
    - Left: date (text-xs text-gray-400) + round badge + amount (text-sm text-gray-100 tabular-nums)
    - Right: source name (text-xs text-gray-400)
    - Below: ai_summary (text-sm text-gray-300, italic, truncated to 2 lines)
  - No deals state: "No deals recorded for this company yet." (Body, text-gray-400, mt-4)

- Investors section (if any):
  - Section heading: "Known Investors" (Heading, mt-8 mb-4)
  - Flex wrap, gap-2: each investor as a Tremor Badge color="gray"

**Interactions:**
- Watchlist toggle: optimistic update, revert on API error with error toast
- "Show more" description toggle: expand inline, no navigation

### View 4: Trend Charts (/trends)

**Layout:**
- Page title: "Trends" (Display, mb-2)
- Subtitle: "Capital flow and deal volume over time" (Body, text-gray-400, mb-8)
- Two-column layout on lg+, single column on mobile (grid grid-cols-1 lg:grid-cols-2 gap-6)
  - Left: Tremor LineChart — "Capital Raised per Week by Deal Type"
    - X axis: week labels ("Mar W1", "Mar W2", etc.)
    - Y axis: USD millions (formatted as "$XM")
    - 4 lines: vc (blue), ma (violet), crypto (amber), ipo (emerald)
    - Tremor legend below chart
  - Right: Tremor BarChart — "Top Sectors by Deal Count (This Month)"
    - X axis: sector names, rotated 45deg if needed
    - Y axis: deal count (integer)
    - Single bar color: amber-400

**Interactions:**
- Both charts rendered on mount, no user-triggered re-fetch in this phase
- Tremor built-in tooltip on hover (no custom tooltip needed)

### View 5: Watchlist (/watchlist)

**Layout:**
- Page title: "Watchlist" (Display, mb-2)
- Subtitle: "Deals from your pinned companies" (Body, text-gray-400, mb-6)
- Same filter bar structure as Deal Feed (type, sector, geo, amount) — but no date range picker (watchlist shows all time)
- Same Tremor Table as Deal Feed, plus:
  - Extra column at end: "Notes" — text-sm text-gray-300 italic, truncated to 1 line, click to open inline edit
  - Remove button per row: icon-only button (X icon), text-gray-400 hover:text-red-400, rightmost column
- Inline notes edit: clicking note text shows a Tremor TextInput inline, pressing Enter or blur saves via PUT, pressing Escape cancels
- Empty state: "Your watchlist is empty." heading + "Visit a company profile and click 'Add to Watchlist'." body (see copywriting contract)

**Interactions:**
- Remove from watchlist: show confirmation inline ("Remove {Company}? Yes / Cancel") before DELETE — no modal
- Notes save: optimistic update on Enter/blur, revert on error
- Notes cancel: Escape key restores original text

## States Required per View

Each view must implement all 4 states:

| View | Loading | Empty | Error | Populated |
|------|---------|-------|-------|-----------|
| Deal Feed | Spinner in table area | "No deals found" + filter hint | Error banner top of table | Table with pagination |
| Heatmap | Spinner centered in grid area | Grid renders but all cells gray-800 | Error banner | Color-scaled grid |
| Company Profile | Full card skeleton (2 gray boxes) | N/A (404 not found: "Company not found") | Error banner | Full profile |
| Trend Charts | Spinner per chart | Charts render with no data lines | Error banner | Charts with data |
| Watchlist | Spinner in table area | Empty state illustration + body copy | Error banner | Table with notes column |

Loading spinner: Tremor Spinner component or Tailwind `animate-spin` on a circle SVG, centered, text-amber-400.
